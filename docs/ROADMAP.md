# Public Roadmap

This roadmap is intentionally conservative. It should describe direction, not promises.

## Now

- Windows desktop workflow for local-first knowledge compilation
- Better ingest for PDFs, scans, web pages, YouTube, and GitHub-style documentation
- Stronger output generation from compiled notes: reports, slides, PPTX, charts, Canvas
- Better UX around AI Skills, source scoping, and generated output review

## Next

- Continued quality and reliability work for ingest and compile flows
- Better packaging and release hygiene for the Windows app
- Clearer onboarding for non-technical users
- Better public product documentation and trust surfaces

## Later

- macOS support
- smoother installer/update experience
- broader public examples and tutorials
- possible release/download improvements if distribution policy changes

## Explicit non-promises

This roadmap does not imply that the following are already shipped:

- team collaboration,
- cloud sync hosted by AlphaTech,
- public source code release,
- native connectors for every external service,
- guaranteed release dates.

## How to read this roadmap

Treat it as product direction. Public-facing copy should stay grounded in what is already shipped or clearly marked as in progress.
